# 火焰纹章风格单关战棋 — 设计文档

日期：2026-06-11
状态：已与用户确认通过

## 1. 目标与范围

用 Python + pygame 实现一个火焰纹章（GBA 时代）风格的**单关战棋演示**：

- 一张 15×10 格手工设计地图（河流+桥、森林带、山地、敌方要塞）
- 我方 4 单位 vs 敌方 6 单位（含 Boss）
- 完整包含三大特色机制：武器克制三角、地形效果、经验升级
- 画面使用 OpenGameArt 的 **DawnLike** 16×16 素材包（CC-BY，需附致谢）

**不在范围内**（明确砍掉，今后可扩展）：多关卡、剧情过场、道具/物品栏、武器耐久、支援对话、职业转职、存档。

## 2. 胜负条件

- **胜利**：歼灭全部敌方单位
- **失败**：主角（领主）阵亡，或我方全灭

## 3. 玩法循环

### 玩家回合
1. 点选我方未行动单位 → 高亮显示移动范围（蓝）与攻击范围（红）
2. 点目标格移动（可取消）
3. 弹出行动菜单：**攻击 / 待机**
4. 选「攻击」→ 选择射程内敌人 → 显示**战斗预测框**（双方伤害/命中/必杀/攻击次数）
5. 确认 → 战斗结算（守方射程内会反击）→ 单位变灰（已行动）
6. 全员行动完毕或手动结束回合 → 敌方回合

### 敌方回合
AI 依次行动：攻击范围内有目标时，选「预期伤害最高、自身受反击风险最低」的目标攻击；否则向最近的我方单位移动。Boss 守在要塞不主动移动，敌人进入其攻击范围才出手。

## 4. 战斗规则（简化 GBA 公式）

### 属性
HP、力量(Str)、技巧(Skl)、速度(Spd)、防御(Def)、移动力(Mov)。

### 结算公式
- 伤害 = 力量 + 武器威力 ± 克制修正(±1) − 守方防御（下限 0）
- 命中率 = 武器命中 + 技巧×2 ± 克制修正(±15) − 守方回避
- 回避 = 守方速度×2 + 地形回避加成
- 必杀率 = 武器必杀 + 技巧/2，必杀伤害×3
- **追击**：速度比对方高 ≥4 时攻击两次
- 攻击顺序：攻方 → 守方反击（若存活且射程内）→ 追击方补刀

### 武器三角
剑 → 斧 → 枪 → 剑（箭头指向被克制方）。克制方 +1 伤害 +15 命中，被克 −1 伤害 −15 命中。弓与魔法不参与三角。

### 射程
剑/斧/枪：1 格；弓：仅 2 格；魔法：1–2 格。

### 地形

| 地形 | 回避加成 | 移动消耗 | 备注 |
|------|---------|---------|------|
| 平原 | 0 | 1 | |
| 森林 | +20 | 2 | |
| 山地 | +30 | 3 | 骑兵不可进入 |
| 水域 | — | 不可通行 | |
| 桥 | 0 | 1 | |
| 要塞 | +20 | 1 | 回合开始回 20% HP |

### 经验与升级
- 攻击命中 +10 exp，击杀 +30 exp，击杀 Boss +60 exp
- 满 100 exp 升级：HP/力量/技巧/速度/防御各按职业成长率（%）独立随机 +1
- 升级弹窗逐项展示成长结果（经典「吓一跳」画面）

## 5. 单位阵容

**我方**：
| 单位 | 职业 | 武器 | 特点 |
|------|------|------|------|
| 主角 | 领主 | 剑 | 均衡，阵亡即败北 |
| 骑士 | 重骑士 | 枪 | 高移动力（骑马），山地不可入 |
| 弓手 | 弓兵 | 弓 | 2 格射程，不被近战反击 |
| 法师 | 魔道士 | 魔法 | 1–2 格射程，打防御低伤害高 |

**敌方**：斧兵×3（被主角的剑克制、克制骑士的枪，用于教学克制）、枪兵×1、弓手×1、Boss（斧，守要塞，属性强）。

## 6. 技术架构

```
火焰纹章/
  main.py                # 入口 + pygame 主循环
  settings.py            # 全部数值常量：属性表、成长率、地形表、颜色、尺寸
  game.py                # 游戏状态机与回合调度
  unit.py                # Unit 类：属性、经验、升级
  combat.py              # 战斗结算纯逻辑（零 pygame 依赖）
  grid.py                # 地图/地形/Dijkstra 移动范围/攻击范围
  ai.py                  # 敌方 AI 决策
  ui.py                  # 高亮、行动菜单、预测框、升级弹窗、信息栏
  assets.py              # DawnLike 切图加载与精灵映射（缺素材时给中文报错）
  tools/fetch_assets.py  # 一键下载解压 DawnLike 到 assets/
  assets/                # 素材目录（gitignore）
  tests/                 # pytest 单元测试
  CREDITS.txt            # DawnLike 致谢（DragonDePlatino, DawnBringer）
```

- **状态机**：`SELECT → MOVE → MENU → TARGET → COMBAT → (升级弹窗) → ...→ ENEMY_TURN`，由 game.py 驱动
- **逻辑与渲染分离**：combat.py / grid.py / unit.py / ai.py 不 import pygame，可独立单测
- **渲染**：16×16 素材 ×3 放大 = 48px 格子；窗口 720×580（地图区 720×480 + 底部信息栏 100px）
- 敌我用色调区分（我方蓝色调、敌方红色调，通过着色处理同一套精灵）

## 7. 错误处理

- 启动时检查 assets/ 是否就绪，缺失则打印中文指引：`请先运行 python tools/fetch_assets.py`
- fetch_assets.py 校验下载完整性（zip 可解压、关键 PNG 存在），失败时给出手动下载地址
- 素材中个别图块缺失时回退为纯色方块渲染，不崩溃

## 8. 测试策略

- **pytest 单测**：combat（克制加成、伤害下限、追击判定、必杀、经验计算）、grid（移动范围含地形消耗、骑兵山地禁入、攻击范围）、unit（升级成长边界）
- **手动验收**：完整游玩一局，覆盖胜利与失败两种结局
